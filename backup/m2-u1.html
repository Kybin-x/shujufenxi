数据采集：电商运营的"原材料"

基础数据采集 · 模块二 · 单元一 数据采集认知

⏱ ~5 min
// OPENING STORY

没有数据，运营就是瞎猜

两家同类目的女装店，A店老板每天凭感觉补货、凭经验选款；B店运营每周采集行业销量数据、竞品价格、自店访客来源，用数据决策选款和定价。

三个月后，A店积压了大量滞销库存，B店爆款率达到60%。

数据采集，是所有数据分析工作的第一步——也是最关键的一步。没有高质量的数据输入，后续所有分析都是"垃圾进垃圾出"。
📊 本单元核心：概念 · 四原则 · 五方法 · 三步骤
数据采集（也叫数据获取），是指通过在平台源程序中预设工具或程序代码，获取商品状态变化、资金状态变化、流量状态变化、用户行为和信息等数据内容的过程，为后续进行数据分析提供数据准备。
// 定义来源：《电子商务数据分析基础》高等教育出版社
🎯 数据采集在电商中采集什么？
📦 商品状态变化
价格波动、库存变动、新品上架、下架记录
💰 资金状态变化
销售额、退款金额、广告费用、利润数据
📡 流量状态变化
UV、PV、跳失率、来源渠道、峰值时段
👤 用户行为和信息
收藏、加购、评价、复购、停留时长

// PART_02 · KNOWLEDGE

四原则 · 五方法 · 三步骤

掌握数据采集的完整知识体系

⏱ ~20 min
⚖️ 二、数据采集的四大原则（点击展开详解）
⚡
及时性
尽可能获取最新数据，只有最新数据才能发现当前问题并预测变化趋势。
为什么重要？电商行业变化极快，昨天的竞品价格和今天可能完全不同。
实操建议：高频指标（流量/转化率）至少每日采集；行业数据每周更新；月度报告月初采集。
📅
有效性
注意数值期限的有效性，过期数据会导致分析结论失去参考价值。
典型误区：用去年双十一的流量数据来预测今年的库存备货，导致严重偏差。
原则：明确数据的采集周期和有效期，超过有效期的数据需重新采集。
🎯
准确性
确保摘录的数据准确无误，避免数据参与计算时出现较大偏差。
典型错误：把"销售量 12,500"手动录入时写成了"1,250"，导致计算的毛利率相差10倍。
检验方法：采集后通过求和/平均与原始数据比对，发现不匹配立即排查。
⚖️
合法性
只能采集已公布的公开数据或经对方同意的数据，不能使用非法手段获取。
合法来源：平台公开数据、行业报告、经用户授权的问卷
非法行为（违法！）：商业间谍、非法窃取竞争对手内部数据、未经同意购买客户资料
法律后果：违反《反不正当竞争法》《个人信息保护法》，可能面临罚款甚至刑事责任。

技术门槛高
▼

典型场景：ERP系统、订单管理系统的销售数据，直接从数据库中查询导出。

④ 报表采集
小店常用
▼
对于没有统计功能的独立站点（如小型独立外贸网站），每天咨询客户数、订单数等指标没有自动统计，可以通过每日、每周的工作报表手动记录采集。

典型示例：客服每日填写"咨询人数/接待人数/询单人数/销售额"汇总表。

适用场景：新品开发前的需求调研、活动后的用户满意度调研、品牌认知度测量。

// PART_03 · CASE

案例分析：这种采集合法吗？

运用四原则和三步骤分析真实场景

⏱ ~12 min
// CASE STUDY · 教材课后思考题

某电商企业：购买客户资料发营销短信

某电子商务企业为了让更多客户了解并浏览自己的网店，通过购买客户资料的手段获取客户联系信息，并通过短信、邮件等方式对客户进行营销。

问题：这种数据采集方式会不会受到法律保护？请说明原因。
📋 按照数据采集三步骤逐一分析
STEP 1
确定采集范围及人员分工
// 分析目标是否清晰
▼
该企业的采集目标是"客户联系信息"（手机号/邮箱），这一目标本身是合理的——电商企业确实需要联系客户进行营销。

✅ 采集目标明确：获取潜在客户联系方式
❌ 采集途径违规：选择了"购买客户资料"这一违法途径，而非通过合法渠道（用户主动注册、平台授权等）
STEP 2
建立必要的数据指标规范
// 数据来源是否合规
▼
数据采集需要建立数据指标的合法来源规范。客户联系信息属于个人隐私数据，其采集必须满足：

① 用户知情同意（用户主动填写、勾选授权）
② 最小必要原则（只采集业务必需的信息）
③ 来源可追溯（能证明数据的合法来源）

该企业购买的客户资料完全无法满足以上三条要求。
STEP 3
数据检查（合规性维度）
// 合法性原则的检验
▼
数据检查不仅要检查完整性、准确性和规范性，还需要对数据来源进行合规性审查：

✅ 完整性

客户联系信息字段完整（姓名、手机、邮箱）——这点"通过"

❌ 准确性

购买来的数据质量无法保证，可能有大量过期或错误信息

❌ 合规性

数据来源不合法，无法通过合规审查，整批数据应当销毁

⚖️ 法律分析：不受法律保护，且构成违法
根据数据采集的合法性原则，该企业的行为违反了多项法律法规：

① 《个人信息保护法》（2021年）：未经用户同意收集个人信息，且将其用于商业营销，违反"告知-同意"原则，最高可罚5000万元或年营业额5%。
② 《反不正当竞争法》：购买客户资料属于通过非法手段获取竞争优势。
③ 《网络安全法》：非法买卖个人信息，情节严重者构成犯罪。

同时，向未授权用户发送营销短信还违反《通信短信息和语音呼叫服务管理规定》，可被处以行政处罚。
🚫 结论：该数据采集行为不受法律保护，企业存在承担民事责任、行政处罚乃至刑事责任的风险。合法做法是通过用户主动注册、平台导购、合规活动等方式获取经用户授权的联系信息。

// PART_05 · SUMMARY

单元小结 · 知识地图

概念 · 四原则 · 五方法 · 三步骤 — 全掌握

⏱ ~5 min
📖一、数据采集的概念
通过预设工具或程序代码，获取商品、资金、流量、用户行为等数据，为后续分析提供数据准备。

也叫数据获取，是数据分析流程的第2步。
⚖️二、四大原则
  • 及时性：采集最新数据，及时发现问题
  • 有效性：注意数值期限，过期数据无效
  • 准确性：确保数据准确，避免计算偏差
  • 合法性：只用公开或授权数据，不违法
🛠️三、五种采集方法
  • 🌐 网页数据采集：爬虫/手动摘录
  • 📋 系统日志采集：服务器访问日志
  • 🗄️ 数据库采集：ERP/订单系统直连
  • 📊 报表采集：日报/周报手动汇总
  • 📝 调查问卷：用户需求/反馈调研
🪜四、三大采集步骤
  • ①确定采集范围及人员分工：明确指标/时间/途径/人员
  • ②建立必要的数据指标规范：唯一标识，贯穿查询、分析、应用全流程
  • ③数据检查：完整性 + 准确性 + 规范性

📌 KEY POINTS — 易错点提醒

⚠️ 合法性是红线

买卖个人信息、非法爬取平台数据、商业间谍——这些都是违法行为，不只是"不合规"，而是有法律风险的。

🔄 方法要匹配场景

用户反馈用问卷；竞品数据用网页采集；自店内部数据用数据库或报表采集——方法选对，事半功倍。

📐 规范先行

数据采集前必须建立指标规范（唯一标识），否则多人采集的数据无法合并分析，造成"数据孤岛"。

🔍 采集完必须检查

完整性、准确性、规范性三维检查，缺一不可。很多数据分析错误不是分析方法的问题，而是采集阶段就出了偏差。
