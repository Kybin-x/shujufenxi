一、数据清洗 — 5类脏数据处理
将脏数据表变成干净可用的数据表
💼 应用场景导入
「你收到了一份从平台导出的订单数据，打开一看……」
有空白的客户姓名、日期格式写着"2024年1月3日"和"Jan 3, 2024"两种格式混用、有几条明显数量超限购的异常订单、还有两条完全相同的重复订单、以及一条TEST测试账号的订单……这些都要先处理掉才能分析！
👁️
先看一眼：数据清洗的5类问题

① 缺失值 ② 格式/内容问题 ③ 逻辑错误 ④ 重复数据 ⑤ 无价值数据

① 缺失值清洗 — 有空格怎么办？必掌握
⚠️ 清洗前（脏数据）
C011 | （空白） | 2024/1/8 | ¥399
C012 | 刘婷婷 | 2024/1/8 | ¥1199
C020 | 邓慧 | （空白） | ¥1299
✅ 清洗后
C011 | 待补充 | 2024/1/8 | ¥399
C012 | 刘婷婷 | 2024/1/8 | ¥1199
C020 | 邓慧 | 已删除/标注 | ¥1299

三种处理方式：① 数据补齐（知道正确值就补）② 删除记录（整行没有分析价值就删）③ 不处理（暂时无法获取，保留空值）

1
定位所有空值
选中数据区域 → 开始 → 编辑 → 查找和选择 → 定位条件 → 选「空值」→ 确定
（所有空白单元格会被同时选中，高亮显示）
2
批量填充
直接输入补充内容 → 按 Ctrl+Enter（批量填充所有选中的空格）
② 格式内容清洗 — 格式乱了怎么统一？必掌握
⚠️ 清洗前（格式混乱）
2024年1月9日（中文格式）
Jan 10, 2024（英文格式）
138 1234 5678（电话含空格）
=H12/E12 → 结果显示#DIV/0!
✅ 清洗后
2024/1/9（统一格式）
2024/1/10（统一格式）
13812345678（空格已删除）
=IFERROR(H12/E12,"/")→显示"/"

📋 日期格式统一操作：

选中日期列 → 右键 → 设置单元格格式 → 数字 → 日期 → 选择统一格式（如 2024/1/1）→ 确定

📋 去除空格操作：

开始 → 编辑 → 查找和选择 → 替换
查找内容：输入一个空格 → 替换为：不输入（空白）→ 全部替换
③ 逻辑错误清洗 — 数据不合理怎么发现？必掌握
⚠️ 逻辑错误示例
数量：5件（限购2件）
支付金额：-129元（负数？）
收货日期早于下单日期
数量：0件（什么都没买？）
✅ 用条件格式标注
数量>2 → 标红提醒
支付金额<0 → 标红提醒
收货<下单 → 标红提醒
人工核查后处理

📋 条件格式标注错误数据：

开始 → 样式 → 条件格式 → 突出显示单元格规则
• 数量>1（限购）：选「大于」→ 填入 1 → 设置红色填充
• 金额<0：选「小于」→ 填入 0 → 设置红色填充
• 收货早于下单：选「其他规则」→ 使用公式：=M2<J2
④ 重复数据清洗 — 一键删重复必掌握
⚠️ 重复数据示例
FY001 | 张小明 | P001 | ¥299
FY001 | 张小明 | P001 | ¥299（完全重复！）
FY009 | 吴雪 | P009 | ¥1299
FY009 | 吴雪 | P009 | ¥1299（重复！）
✅ 删除后
FY001 | 张小明 | P001 | ¥299
FY009 | 吴雪 | P009 | ¥1299
（重复行已删除，保留唯一值）
1
打开删除重复项
选中数据区域 → 数据 → 数据工具 → 删除重复项
2
选择判断重复的列（很重要！）
勾选"订单编号"（只看订单号相同才算重复）
⚠️ 如果同时勾选所有列，只有每一列都相同才算重复，要根据实际情况判断
3
确认结果
点击确定 → Excel显示：删除了X个重复值，保留了X个唯一值
⑤ 无价值数据清洗 — 要不要删？了解
⚠️ 无价值数据示例
TEST000001 | 测试账号 | 测试商品 | ¥0.01
（这是开发测试时产生的，对分析无意义）
✅ 处理建议
先备份整张表（另存一份）
再删除无价值字段/记录
通常不建议直接删除！
⚠️
重要提醒：不建议直接删！

删除前务必备份。今天"没用"的数据，可能明天就需要用到。建议另存备份后再删除，或者用颜色标注而非直接删除。

🧠 记忆口诀
缺失补充或删除，格式统一空格清
逻辑错误条件标，重复数据一键删
无价值数据先备份再动手

专项讲解：IFERROR 错误处理函数
让表格里的#DIV/0! #VALUE! #N/A 等丑陋错误值消失
👁️
先看一眼：IFERROR做什么？

如果公式正常就显示计算结果；如果公式出错就显示你指定的内容（如"-"或"数据缺失"），让表格更干净。

// IFERROR 格式
=IFERROR(原来的公式, 出错时显示的内容)
❌ 不用IFERROR的问题
=H12/E12 → 当E12=0时，显示 #DIV/0!

整张表出现红色错误值，报告难看，函数也无法继续计算

✅ 用IFERROR包裹后
=IFERROR(H12/E12, "/") → 出错时显示 "/"

表格干净，用"/"或"数据缺失"等友好文字替代错误值

📍 3个常见应用场景
场景①：除法公式防止除以零错误
=IFERROR(支付买家数/访客数, 0)
场景②：VLOOKUP找不到时防止#N/A
=IFERROR(VLOOKUP(H4,$A$4:$E$18,2,0), "商品不存在")
场景③：数据类型不匹配时防止#VALUE!
=IFERROR(LEFT(A1,5), "格式异常")

二、数据转化
行列互换 · 类型转换 · 文本提炼
👁️
先看一眼：数据转化做什么？

对数据的格式或结构进行转换，是数据处理的前期准备。让数据从「不好用」的格式变成「好用」的格式。

转化①：数据表行列互换（转置）

💼 场景
「这张表的行太多了，我想把行和列对换，让数据横向展示。」
方法一：粘贴转置
选中数据 → Ctrl+C → 点击目标单元格 → 开始 → 粘贴 → 转置
方法二：快捷键
复制后按 Ctrl+Alt+V → 弹出「选择性粘贴」→ 勾选「转置」→ 确定

转化②：数值与文本类型互换（解决科学计数法）

⚠️ 问题场景
订单号显示为科学计数法：5.7E+17
文本型数字无法用SUM计算
✅ 解决方法：分列
数据 → 分列 → 下一步→下一步
列数据格式选「文本」→完成（解决科学计数）
或选「常规」→完成（文本转数值）
💡
分列法的妙用

不仅可以转换类型，还可以把"20240101"这样的文本转成真正的日期格式！第3步选「日期」→「YMD」→完成

✂️
文本函数：LEFT / MID / RIGHT
从混合数据中提炼出有用信息
💼 应用场景导入
「客户信息全挤在一个单元格里：姓名+手机号+订单号，怎么把它们分开？」
原始数据：wen小文15191842599569834882682982536
需要分别提取：客户姓名 / 联系电话 / 订单号，分别放到不同列
👁️
先看一眼：三个文本函数的区别

LEFT=从左边截取 · MID=从中间截取 · RIGHT=从右边截取

LEFT从左截取
从文本左侧提取指定个数的字符
=LEFT(text, num_chars)
text：要提取的原始文本（单元格）
num_chars：从左边数，要提取几个字符
📍 电商场景：
原始：A1="wen小文15191842599..." (客户名"wen小文"占5个字符)
=LEFT(A1, 5) → 结果：wen小文
📍 提取商品分类前缀：
=LEFT(B2, 2) → 从"数码配件-耳机"中提取"数码"
MID从中间截取
从文本中间指定位置提取指定个数字符
=MID(text, start_num, num_chars)
text：原始文本
start_num：从第几个字符开始提取
num_chars：提取几个字符
📍 电商场景（续上例）：
前5个字符是姓名，第6到16位是手机号（11位）
=MID(A1, 6, 11) → 结果：15191842599
📍 提取订单日期（如订单号"20240601001"第1到8位是日期）：
=MID(B2, 1, 8) → 结果：20240601
RIGHT从右截取
从文本右侧提取指定个数的字符
=RIGHT(text, num_chars)
text：原始文本
num_chars：从右边数，要提取几个字符
📍 电商场景（续上例）：
订单号固定18位，在混合字符串的最后18位
=RIGHT(A1, 18) → 结果：569834882682982536
📍 提取快递单号后8位：
=RIGHT(C2, 8) → 从"SF1234567890"中取最后8位
🧪 综合案例完整公式

原始数据A1："wen小文15191842599569834882682982536"

// B列：提取客户姓名（前5个字符）
=LEFT(A1, 5)
// C列：提取手机号（第6位开始，11个字符）
=MID(A1, 6, 11)
// D列：提取订单号（最后18位）
=RIGHT(A1, 18)
🧠 记忆口诀
LEFT从左MID中间，RIGHT从右来截取
参数一个是位置，另一个是字符数

三、数据排序 — 3种排序方式
简单排序 · 高级排序 · 自定义排序 + 重要避坑！
👁️
先看一眼：3种排序方式

简单排序=单列快速排 · 高级排序=多列排（主+次关键字）· 自定义排序=按照你规定的顺序排

① 简单排序（单列快速排）

1
选中要排序的列
点击「支付金额」列中的任意单元格
2
点击升序或降序
数据 → 排序和筛选 → 点击「降序」（金额从高到低）
⚠️
关键！弹出排序提醒时必须选「扩展选定区域」！
❌ 「以当前选定区域排序」
只有选中的列被重新排序，其他列保持原位！导致数据错位！客户A的金额跑到客户B那行去了！
✅ 「扩展选定区域」
整行数据一起排序，每行的数据保持完整对应关系，不会错位

② 高级排序（多列多条件）

💼 场景
「先按访客数降序，访客数相同的再按支付金额降序」
1
打开排序对话框
数据 → 排序
2
设置主要关键字
主要关键字：选「访客数」→ 顺序：降序
3
添加次要关键字
点击「添加条件」→ 次要关键字选「支付金额」→ 降序 → 确定

③ 自定义排序（按自定规则排）

💼 场景
「我想按流量来源的重要性排序：付费流量→自主访问→淘内免费→淘外网站→其他，而不是按字母顺序」
1
创建自定义序列
文件 → 选项 → 高级 → 常规 → 编辑自定义列表 → 输入：付费流量,自主访问,淘内免费,淘外网站,其他来源 → 添加 → 确定
2
排序时选择自定义序列
数据 → 排序 → 主要关键字「流量来源」→ 次序选「自定义序列」→ 选已定义好的序列 → 确定
🧠 记忆口诀
排序三类要记牢：简单高级自定义
弹出提示必扩展，否则数据会错位
