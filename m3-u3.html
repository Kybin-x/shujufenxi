小林已经学会了分类统计的四种方法，准备大展身手。店长又发来一份流量推广数据——2000多行，小林信心满满打开，准备直接做数据透视表，却发现问题一个接一个：

😱
问题一个接一个：第1-2行完全相同（重复数据）、第3行访客数空缺（缺失值）、第4行支付金额为负数（逻辑错误）、第5行日期格式不统一（格式问题）、第6行下单数是汉字（格式问题）……用这样的数据做分析，结果一定是错的！
小林意识到：在分析数据之前，必须先把数据"洗干净"、"转规范"、"排整齐"。这就是本单元要学习的三项核心操作——数据清洗、数据转化和数据排序。

数据处理三项核心操作总览
处理操作目的通俗理解
🧹 数据清洗提升数据质量，确保准确性、完整性和一致性把"脏东西"挑出来——删多余的、补缺少的、改错误的
🔄 数据转化将数据的格式或结构转换为适合分析的形式把"不合身的衣服"改一改——换格式、拆信息、调结构
📶 数据排序按一定规则排列数据，便于查看和后续操作把"乱放的东西"摆整齐——按大小排、按类别排、按自定义顺序排
💡实际工作中的一般顺序：先清洗 → 再转化 → 最后排序。清洗是基础——不把错误和重复数据处理掉，后续转化和排序都没有意义。
五种常见数据质量问题
1
缺失值
某单元格没有数据，是空的
如：访客数未填写
2
格式内容问题
格式不统一或含多余空格
如：日期格式混用
3
逻辑错误
数据不符合常理或规则
如：支付金额为负数
4
重复数据
同一条记录录入了多次
如：同一订单出现两行
5
无价值数据
对当前分析无用的字段
如：客户备注、物流单号
1️⃣
缺失值清洗
处理方式适用场景举例
数据补齐能通过其他途径获取正确数据时访客数为空，从平台后台重新导出补上
删除记录缺失信息无法获取，且该条记录不影响整体分析某条订单所有关键字段都为空，直接删除整行
不处理缺失字段不影响当前分析目标分析销售额时，"客户备注"为空不影响计算

2️⃣
格式内容清洗
（1）日期格式不统一
原始数据状态
2024/3/1✅ 标准格式
2024年3月2日❌ 中文格式，不统一

（2）文本中含多余空格

（3）单元格中的错误标识符
错误标识常见原因通俗理解解决方法
#####列宽不够，数据显示不下"格子太窄了"拖宽列宽即可
#DIV/0!公式中除数为零或为空"不能除以0"检查被除的单元格是否为空或0
#VALUE!数据类型不匹配"用文字做加减法算不了"检查参与计算的单元格是否含文本
用 IFERROR 函数优雅处理错误值
原公式 vs 改进公式
// 原公式：除数可能为0时会报错
=H2/E2 → 当 E2=0 时显示 #DIV/0!
// 改进公式：出错时显示"/"代替错误代码
=IFERROR(H2/E2, "/")
公式正常 → 显示计算结果（如 3.04%）；公式出错 → 显示 /
💡修改一个单元格的 IFERROR 公式后，拖动右下角的填充柄（小方块）即可批量应用到其他行。

无价值数据清洗
原则说明
⚠️ 先备份，再删除删除前一定要保存一份原始数据的备份，防止误删

操作二：文本数据提炼（LEFT/MID/RIGHT）
📋
真实场景
某系统导出的客户信息把所有内容塞进一个格子：张小文15191842599山西运城569834882682982536
姓名、电话、地址、订单号全部挤在一起，无法直接分析，需要逐项拆分提取。
LEFT
从文本左边提取指定个数的字符
=LEFT(文本, 字符数)
💡 从左数，取几个字
MID
从指定位置开始提取指定个数的字符
=MID(文本, 起始位置, 字符数)
💡 从第几个字开始，取几个字
RIGHT
从文本右边提取指定个数的字符
=RIGHT(文本, 字符数)
💡 从右数，取几个字
A1 = "张小文15191842599山西运城569834882682982536"
// 提取姓名（最左边3个字）
=LEFT(A1, 3) → 张小文
// 提取电话（第4个字符开始，共11位）
=MID(A1, 4, 11) → 15191842599
// 提取订单号（最右边18位）
=RIGHT(A1, 18) → 569834882682982536
💡确定 MID 的"起始位置"：数一数目标内容前面有几个字符，加1就是起始位置。电话前面有"张小文"3个字，所以起始位置 = 3+1 = 4。

自定义排序

按特定的顺序排列数据，而不是简单升序/降序。例如：把流量来源按重要性排列：自主访问 → 付费流量 → 淘内免费 → 淘外网站 → 其他来源。

1

2
使用自定义序列排序
「数据」→「排序」→ 主要关键字选"流量来源" → 在「次序」下拉选「自定义序列」→ 选择刚才创建的序列 →「确定」
