3.3.7 数据类型转换

解决文本型数字无法求和、科学计数法显示错误等典型格式障碍
📁 配套文件：3.3.7_练习数据.xlsx 🎯 三个难度等级 🏢 情景：星途电商有限公司 (ERP数据清洗)

📋 背景情景

星途电商有限公司的业务数据存放在旧版 ERP 系统中。由于系统导出机制的问题，下载到 Excel 的数据往往存在"类型错位"：销售额变成了无法计算的文本（带绿三角），长订单号变成了看不懂的科学计数法（1.23E+10），日期也被识别成了普通文本。这些问题直接导致后续的 SUM 求和结果为 0。请协助完成数据标准化转换。

📖 知识点速览：常见类型障碍

1. 文本型数字 (带绿三角)
❌ 表现：单元格左上角有绿三角，SUM求和为0
✅ 修复：点感叹号选"转换为数字"或使用【分列】
2. 科学计数法 (1.2E+10)
❌ 表现：超过11位数字自动变形，末尾变0
✅ 修复：选中→设置单元格格式→数字(0小数)或特殊(邮政编码)
3. 文本型日期
❌ 表现：无法进行日期排序或按月透视
✅ 修复：使用【分列】功能，在第三步选"日期:YMD"
LEVEL 1

基础练习

25条订单 · 处理销售额文本型数字
📊 基础数据：销售额列为文本型，验证SUM结果。
✅ 任务要求
  1. 将数据粘贴到 Excel，在销售额底部（C27）尝试输入 =SUM(C2:C26)，观察结果是否为 0。
  2. 选中 C 列销售额数据，点击单元格旁的黄色感叹号图标，选择"转换为数字"。
  3. 再次观察 C27 的 SUM 结果，确认是否恢复正常。
  4. 记录转换前后的求和数值。

LEVEL 2

巩固练习

28条数据 · 文本数字 + 科学计数法 + 文本日期
📊 巩固数据：含订单号(变形)、销售额(文本)、下单日期(文本)。
✅ 任务要求
  1. 修复订单号：A列订单号显示为 1.23E+11，请通过"设置单元格格式"恢复其 12 位数字显示。
  2. 修复销售额：使用 VALUE 函数在 D 列建立新列转换 C 列的文本数字。
  3. 修复日期：选中 E 列日期，通过【数据】-【分列】功能，在第三步选择"日期：YMD"，将其转为标准日期。
  4. 完成下表，验证所有列是否转换成功。
💡 提示：科学计数法若数值已经丢失（末尾变0），设置格式也无法找回。但在本练习数据中，仅为显示问题，可直接恢复。
LEVEL 3

进阶练习

30条 ERP 原始数据 · 混合障碍转换 (下调难度版)
📊 进阶数据：模拟 ERP 导出，混合了文本数字与文本日期障碍。
✅ 任务要求
  1. 识别障碍：使用 ISNUMBER 函数检测销售额列，记录有多少行是 FALSE（文本）。
  2. 分列转换法：对销售额和下单日期两列，分别使用【分列】工具完成类型标准化。
  3. 逻辑验证：转换后，计算"日均销售额"，若日期和金额任一类型错误，公式将报错。
  4. 撰写说明：在 Excel 中简述为何"分列"比手动一个一个点绿三角转换更高效。
💡 技巧：分列功能（Data -> Text to Columns）不仅能拆分文字，其第三步的"列数据格式"是批量转换数据类型的"万能钥匙"。
