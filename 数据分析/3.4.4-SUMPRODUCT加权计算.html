3.4.4 SUMPRODUCT 加权计算

用加权综合评分客观比较多维度数据，掌握 SUMPRODUCT 函数的核心用法
📁 配套文件：3.4.4_练习数据.xlsx 🎯 三个难度等级 🏢 情景：星途电商有限公司

📋 背景情景

星途电商有限公司运营总监需要对各推广渠道和商品进行综合评估， 以便决定预算分配和选品策略。但各渠道/商品在不同维度（ROI、转化率、销量等）上的表现各有高下， 单看某一指标容易以偏概全。运营分析师小林决定采用加权综合评分模型—— 先确定各维度的权重（重要程度），再用 SUMPRODUCT 函数快速算出每个对象的综合得分， 最终按分数排名，为决策提供客观依据。

📖 核心概念：加权平均分 与 SUMPRODUCT

加权综合评分：对多个维度分别打分（通常 1-10 分），再乘以各自权重后求和，得到综合得分。 权重代表该维度的重要程度，所有权重之和 = 1（100%）。

❌ 手动逐项相乘再相加（繁琐）
= B2×0.4 + C2×0.3 + D2×0.2 + E2×0.1
有几个维度就要写几个乘法，维度多时公式极长，且更改权重麻烦
→
✅ SUMPRODUCT 一步完成（简洁）
= SUMPRODUCT( B2:E2, $B$1:$E$1 )
将评分行与权重行对应相乘后求和，修改权重只需改第1行，所有得分自动更新

SUMPRODUCT 语法：=SUMPRODUCT(数组1, 数组2) — 将两个大小相同的区域对应位置相乘，然后把所有乘积加总。
要求：两个数组的行数和列数必须完全一致。

⚠️ 权重行必须用绝对引用（如 $B$1:$E$1）， 这样公式向下填充时权重行不变，评分行随行号变化。

✅ 任务要求
  1. 将数据粘贴到"数据（基础）"Sheet，确认第1行为权重行（B1=0.4，C1=0.35，D1=0.25）。
  2. 在 E2 单元格用手动逐项相乘的方式写出综合得分公式：
    =B2*0.4 + C2*0.35 + D2*0.25，向下填充到 E5。
  3. 在 F2 单元格改用 SUMPRODUCT 函数写出综合得分公式：
    =SUMPRODUCT(B2:D2, $B$1:$D$1)，向下填充到 F5。
  4. 对比 E 列和 F 列的结果，验证两种方法是否一致（应完全相同）。
  5. 在 G 列用 RANK 函数对综合得分排名：=RANK(F2, $F$2:$F$5, 0)。
  6. 完成后回答分析问题。
🔧 SUMPRODUCT 公式关键点
  1. 评分区域：用相对引用，如 B2:D2（随行变化）
  2. 权重区域：用绝对引用，如 $B$1:$D$1（始终是第1行）
  3. 两个区域大小必须相同：3列对3列
  4. 向下填充时，评分区域的行号自动变（B2→B3→B4→B5），权重区域不变

💡 操作提示
  • RANK 函数语法：=RANK(数值, 引用区域, 0) — 最后的参数 0 表示降序（最大值排第1），1 表示升序。
  • 修改权重验证时，把 B1 改为 0.25、D1 改为 0.40（确保合计仍为 1）， F 列的 SUMPRODUCT 得分会自动重新计算。
  • 两列结果完全一致说明 SUMPRODUCT 理解正确；若有差异，检查绝对引用是否正确。
  • 综合得分保留 2 位小数，方便对比细微差异。

💡 操作提示
  • 权重合计验证：在权重行旁边用 =SUM(B1:E1) 确认权重之和为 1.00， 否则综合得分无意义。
  • 权重设定参考：主营高利润商品时，利润率权重可设 0.40-0.50； 关注市场扩张时，销量潜力权重较高；竞争激烈赛道需重点考虑竞争强度得分。
  • 敏感性测试操作：直接修改第1行的权重数值（SUMPRODUCT 结果自动更新）， 记录排名变化后，再改回原值。建议在操作前用备忘录记录原始权重。
  • 综合得分越高并不意味着一定优先推广，还需结合店铺实际库存、资金等情况综合判断。

🔒 进阶权重设计约束条件
  • 所有权重之和 = 1（100%）
  • 利润率权重不能超过 40%（避免过度追求利润忽视其他维度）
  • 客户评价权重不能低于 10%（客户口碑是长期运营的基础）
  • 每个维度权重至少 5%（不能将某维度完全忽略）
  • 在满足以上约束的前提下，根据店铺策略自由设计权重
✅ 任务要求
  1. Step 1：处理缺失评分。 部分新品的历史数据维度为空，处理规则：
    • 销量潜力为空 → 用该列所有有数据的行的均值填充（可用 AVERAGE 计算后手动填入）
    • 退货率为空 → 填入 5（中间值，表示"未知，按中等处理"）
    • 客户评价为空 → 填入 5（评价量不足，暂按中等计）
    填充完成后，原空单元格标注黄色背景，提示为推算值。
  2. Step 2：设计权重。 在满足约束条件的前提下，填写第1行权重，并在"权重设计说明"区解释选择理由。
  3. 在 G 列用 SUMPRODUCT 计算综合得分（引用 B-F 列评分和 $B$1:$F$1 权重行）。
  4. 在 H 列用 RANK 排名，在 I 列用 IF 给出初步建议：
    =IF(H2=1,"★重点推广",IF(H2<=3,"建议推广","维持现状"))
  5. 权重约束验证：在权重行旁边写验证公式：
    • 权重合计：=SUM(B1:F1)，结果应为 1
    • 利润率不超标：=IF(B1<=0.4,"✓","⚠️超标")
    • 客户评价达标：=IF(F1>=0.1,"✓","⚠️不足")
  6. 在"选品报告"Sheet 中，按下方模板撰写完整选品报告。

💡 操作提示
  • 销量潜力列均值的计算：=AVERAGE(C2:C9)（只对有数据的行计算，AVERAGE 自动忽略空值）， 然后手动填入空单元格，并将该格背景设为黄色。
  • SUMPRODUCT 包含5列：=SUMPRODUCT(B2:F2, $B$1:$F$1)， 确保评分区域和权重区域都是5列。
  • 约束验证公式：放在权重行旁边（如 G1/H1 等单元格）， 用不同颜色标注"✓达标"或"⚠️超标"，方便一眼看出是否满足约束。
  • 含推算值的SKU得分可信度较低，在报告中应注明"该SKU综合得分含推算值，仅供参考， 建议收集实际数据后重新评估"。
  • 选品报告每项 2-3 句话，重点是数据有依据、建议可操作，不需要长篇分析。
📌 SUMPRODUCT 加权评分模型的使用原则
  • 权重设计要有依据：不同店铺策略决定不同权重，不存在唯一正确的权重，但需要说清楚理由。
  • 评分标准要统一：所有维度使用同样的评分尺度（如1-10），否则不同量级的数据直接乘以权重会失真。
  • 综合得分是辅助决策工具，不是唯一标准；特殊情况（如某商品库存为0）需在报告中单独说明。
  • 定期更新评分：随着市场变化，各维度的实际得分也会变化，建议每季度重新评估。
